## Executive Summary

This report analyzes how the NNIE (Neural Network Inference Engine) face recognition model responds to various image mutations. By enrolling 297 systematically mutated versions of a single face image, we mapped input perturbations to embedding changes.

### Key Findings

1. **High-contrast mutations DESTROY embeddings** - Contrast ratios above 1.37x reduce similarity to ~0.31, essentially creating a different identity
2. **Color channel shifts are devastating** - Red channel +50 drops similarity to 0.31
3. **Model is ROBUST to blur and compression** - Gaussian blur and JPEG quality changes maintain >0.97 similarity
4. **Brightness has moderate impact** - High brightness (>1.6x) can reduce similarity to ~0.87

## Security Implications

### Evasion Attack Potential

An attacker could potentially evade face recognition while remaining visually recognizable by applying:

1. **High contrast (1.5-1.7x)** - Drops similarity to 0.31-0.34
2. **Red color cast** - Channel shift +50 drops to 0.31
3. **Combined mutations** - Some combinations achieve 0.32 similarity

### Model Robustness

The model shows strong robustness to:
- **Gaussian blur** (up to 5.0 sigma) - maintains 0.97+ similarity
- **JPEG compression** (quality 10-95) - maintains 0.99+ similarity  
- **Minor brightness changes** - maintains 0.95+ similarity

## Methodology

1. Generated 393 pixel-level mutations of a single enrolled face
2. Batch imported via DTM-600 web GUI
3. Extracted 297 successful templates via device shell
4. Parsed 256-dimensional float embeddings from binary format
5. Computed cosine similarity to mean embedding
6. Grouped and analyzed by mutation type
